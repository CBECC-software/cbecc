;-------------------------------------------------------------------------
; Table of Building Database Data Types & Interface Flags
;-------------------------------------------------------------------------
; ASHRAE/IES Standard 90.1-1989 (Equivalent) - Energy Cost Budget Method
; Compiled for US DOE by RS_Briggs@pnl.gov
; September 30, 1998
;-------------------------------------------------------------------------
;
; Table Format:
;    <bldg component name>:<parameter name>([<array index>]),
;                      <compliance data type>,
;                      <primary variable>, <user editable>,
;                      <user defined default allowed>,
;                      <display input>, <display proposed>,
;                      <display budget>
;
;    <bldg component name> : name of building component which parameter
;                            describes.
;
;    <parameter name> : name of parameter for which datatype data is provided.
;
;    <array index> : only provided for building parameters which are
;        defined by an array of values.  The array index is either a 1-based
;        index into the array of values, or "ALL" implies the datatype data
;        is applied to all elements in the array.
;
;    <compliance data type> = 
;        Compulsory   : User MUST provide input if the object exists in the proposed design.
;                       The property will be included in the SDD Input Data Model.txt file
;                       In the UI, users will be prompted to define this during object creation.
;                       Compliance analysis cannot be performed until the property is defined.
;        Required     : May be defaulted by the ruleset, but if not the user MUST provide input
;                       for any object of this type in the proposed design.
;                       Compliance analysis cannot be performed until the property is defined.
;        CondRequired : May be defaulted by the ruleset, but if not the user must provide input
;                       for compliance analysis if it is required based on other building data.
;                       Conditional requirements are enforced by errors posted by Model*Check rules.
;                       The property will be included in the SDD Input Data Model.txt file.
;        Optional     : User MAY provide input if it applies to an object in the proposed design.
;                       The property will be included in the SDD Input Data Model.txt file.
;        Default      : An Optional, Required or CondRequired property that is set to a default
;                       value by either BEMEnums or ProposedInput defaulting rule.
;                       The user may override the default value.
;                       The property will be included in the SDD Input Data Model.txt file.
;        CriticalDef  : NOT SUPPORTED: Default value provided which if overridden by user, must
;                       be accompanied by user-defined documentation on why the default
;                       was overridden.
;        Prescribed   : Can be specified by the user, but typically only in a research or testing
;                       mode and will not be included in the SDD Input Data Model.txt file.
;        NotInput     : Either not allowed for the specified ruleset, is an intermediate/calculated
;                       ruleset parameter, or is only used for simulation or reporting purposes.
;                       The property will not be included in the SDD Input Data Model.txt file.
;
;    <primary variable> =
;        = 0 if NOT a primary variable for this building object.
;        = 1 if IS a primary variable for this building object.
;        "Primary" inputs are those inputs which are the dependent variables
;        in expressions which define other Default, CriticalDef or
;        Prescribed variables of this object type.  Whenever a building
;        object is created, the user must first define it's Primary
;        variables.  Once the primary variables are defined by the user, a
;        complete building object dialog is presented which includes ALL
;        variables which describe that building object.
;
;    <user editable> = 
;        = 0 if this variable CANNOT be edited by the user.
;        = 1 if this variable CAN be edited by the user.
;        Note:  This flag is only referenced for those compliance datatypes
;               which are compatible with user input
;               (i.e. Compulsory, Required, Optional, Default and
;                     CriticalDef)
;
;    <user defined default allowed> = 
;        = 0 if this variable CANNOT reference a user defined default value.
;        = 1 if this variable CAN reference a user defined default value.
;        Note:  This flag is only referenced for those compliance datatypes
;               which are compatible with user-defined defaults
;               (i.e. Compulsory, Required, and Optional)
;
;    <display input> => unused for CEC-Residential / CBECC-Res
;
;    <display proposed> => unused for CEC-Residential / CBECC-Res
;
;    <re-classify as user-defined> => controls whether or not rule-defaulted
;        data is re-classified as user-defined and written to project
;        input files with each File-Save operation.
;        = 0 if this variable SHOULD NOT be re-classified as user input
;            upon each File-Save operation (rule defaulting will persist)
;        = 1 if this variable SHOULD be re-classified as user input
;            upon each File-Save operation (subsequent rule defaulting will
;            be disabled)
;        Note: this flag applies only to variables classified as Compulsory,
;        Required, Optional, Default or CriticalDef -AND- have a "user
;        editable" setting of '1'
;
;-------------------------------------------------------------------------
; Special Cases:
;
;   1) Bldg Component Name = DEFAULT  =>  All parameters NOT specified
;      in this datatype.txt file will be assigned the datatype settings
;      defined following the DEFAULT code word.  The DEFAULT datatype entry
;      must come BEFORE all other datatype entries.
;
;   2) For parameters which are described as an array of values, datatype
;      values can be set for the entire array by enclosing the word "ALL"
;      in the square brackets following the component/parameter name.
;      The setting of datatype information is performed in the same order
;      as they are defined in this list.  This means that if the user wished
;      to set all array elements except the first to a particular setting,
;      then they can first specify the datatype for ALL elements in the 
;      array and then respecify the datatype for just the first element.
;      For example:
;           Space:Area[ALL],     Optional,   0,    1,   1,    1,  1,  1
;           Space:Area[1],     Compulsory,   0,    1,   1,    1,  1,  1
;      
;-------------------------------------------------------------------------
